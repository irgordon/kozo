# KOZO Runtime Evidence Review

Version: 1
Status: Authoritative
Scope: Release review gate for current runtime evidence claims, paths, metadata, and non-goals

---

# 1. Purpose

This document defines the release-review gate for KOZO runtime evidence.

The repository already generates runtime smoke evidence. This review gate makes sure release reviewers can tell what that evidence proves, what it does not prove, where it lives, and which validator protects it.

This is not a QEMU boot phase.

This is not hardware trap execution evidence.

# 4. Evidence Type

Evidence type: `runtime-adjacent-object-symbol-smoke`

This evidence checks object and symbol surfaces for the current kernel build path.

It is runtime-adjacent evidence because it inspects generated objects and symbol/string evidence. It is not emulator boot evidence.

---

# 5. Evidence Files

Live evidence log: `artifacts/runtime/runtime_smoke.log`

Live metadata: `artifacts/runtime/runtime_smoke.metadata.json`

Release bundle target:

Release evidence log: `artifacts/release/runtime/runtime_smoke.log`

Release metadata: `artifacts/release/runtime/runtime_smoke.metadata.json`

```text
artifacts/release/runtime/runtime_smoke.log
artifacts/release/runtime/runtime_smoke.metadata.json
```

The live files are regenerated by `scripts/runtime_smoke.sh` and `scripts/verify.sh`.

Full CI requires these live files because it runs `scripts/verify.sh`.

The lint workflow does not require these live files unless it is changed to run full verification.

The release bundle target is a copied review artifact for release review.

The release bundle copy does not become source truth.

---

# 6. Required Validator

Validator: `runtime_smoke_evidence`

The full verification harness must also include `runtime_evidence_review`.

`runtime_smoke_evidence` protects the generated runtime smoke log and metadata.

`runtime_evidence_review` protects release-review wording, non-goals, release references, and metadata alignment.

---

# 7. What The Evidence Proves

This evidence proves:

* the freestanding x86_64 kernel object build path succeeds
* x86_64 boot and syscall assembly objects can be assembled
* runtime entry and dispatcher symbols are present in binary evidence
* syscall bridge symbols are present in binary evidence
* current serial heartbeat marker strings are present in binary evidence
* the generated runtime smoke artifact is available for release review

# 9. Review Checklist

Before release review:

* Confirm `scripts/verify.sh` passes.
* Confirm `runtime_smoke_evidence` passes.
* Confirm `runtime_evidence_review` passes.
* Confirm `artifacts/runtime/runtime_smoke.log` exists.
* Confirm `artifacts/runtime/runtime_smoke.metadata.json` exists.
* Confirm metadata `evidence_type` is `runtime-adjacent-object-symbol-smoke`.
* Confirm live evidence and metadata paths are referenced by release evidence policy.
* Confirm release bundle target paths are documented.
* Confirm metadata `does_not_prove` includes all required runtime non-goals.
* Confirm release notes do not claim QEMU boot, hardware trap execution, Linux compatibility, userspace execution, production readiness, or unproven subsystem behavior.

---

# 10. Release Blocker Rules

Release review is blocked if runtime evidence is missing.

Release review is blocked if runtime evidence metadata is missing or invalid.

Release review is blocked if runtime evidence is described as QEMU boot evidence.

Release review is blocked if runtime evidence is described as hardware syscall/trap execution evidence.

Release review is blocked if required runtime non-goals are missing.

Release review is blocked if release notes overclaim runtime evidence.

Release review is blocked if `runtime_smoke_evidence` or `runtime_evidence_review` fails.
